  • 在方法上设置异常的抛出管道

抛出多个异常

异常的注意事项

  • 异常处理只有两种方式：catch和throws

  • 异常处理，必须二选一

  • Java语法强制开发者事先考虑异常如何处理

  • 初级开发者，可能会捕获，但不处理异常

  • 方法重写时，异常管道不能比父类方法多

如何选择

  • 底层异常，应该向前抛到前面处理
  • 经验少时不知道该在什么位置捕获处理，应该选择throws

throw

  • 手动抛出异常，执行异常的抛出动作
  • 类比成return

  • 异常简化，把多种类型异常，简化成一种
